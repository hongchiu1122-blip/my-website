1. 注册 Apple Developer Program 账号

在一切开始之前，你需要成为 Apple Developer Program 会员。只有付费会员才能将 App 发布到 App Store。

账号类型对比

类型个人 / 独资企业组织
费用$99 USD / 年（非营利组织、认证教育机构、政府实体可申请费用豁免）
卖家名称显示个人法律姓名显示组织法律名称
需 D-U-N-S否是（邓白氏编号，用于验证组织身份）
团队协作不可分配角色可邀请成员、分配角色权限
注册要求Apple ID（开启双重认证）+ 法定年龄 + 真实姓名地址（不接受邮箱信箱）Apple ID（开启双重认证）+ 法定实体 + D-U-N-S + 工作邮箱（关联组织域名）+ 公开可访问的官网

1注册步骤

  1. 准备一个 Apple ID，并在「设置 → Apple 账户」中开启双重认证（2FA），这是硬性要求。
  2. 前往 developer.apple.com/programs/enroll，点击「Start Your Enrollment」。
  3. 选择账号类型：Individual（个人）或 Organization（组织）。
  4. 个人：填写真实法定姓名、邮箱、电话、地址（P.O. Box 信箱不被接受）。
  5. 组织：填写组织法律实体名称、D-U-N-S 编号、工作邮箱、官网网址。注册人须为组织所有者/高管/高级项目负责人，或经授权的员工。
  6. 提交后等待 Apple 身份验证（个人通常 1-2 天，组织需验证 D-U-N-S 可能更久）。
  7. 验证通过后，同意《Apple Developer Program 许可协议》并完成 $99/年付款。
  8. 付款成功后即可登录 App Store Connect 和 开发者账户后台。
⚠️ 注意组织账号的 D-U-N-S 编号可免费申请，但 Dun & Bradstreet 审核可能需要数天到数周。建议尽早申请。组织名必须与营业执照完全一致。
来源：Apple Developer Program — Become a member

2. 配置证书、App ID 与描述文件

iOS 上架需要一套签名凭证。这套东西在 Apple Developer 后台的「Certificates, Identifiers & Profiles」中管理。

三个核心概念

Certificate 证书
证明你的开发者身份。发布用 Distribution Certificate (App Store)，开发调试用 Development Certificate。
Identifier (App ID)
应用的唯一标识，对应 Bundle ID（如 com.yourcompany.yourgame）。需开启所需 Capabilities。
Provisioning Profile
把「证书 + App ID + 设备」绑定在一起的描述文件。发布用 App Store Provisioning Profile。

2配置流程

  1. 登录 developer.apple.com/account → Certificates, Identifiers & Profiles。
  2. 创建证书：Certificates → 「+」→ 选 Apple Distribution → 上传 CSR 文件（在 Mac「钥匙串访问」中生成）→ 下载并双击安装 .cer。
  3. 注册 App ID：Identifiers → 「+」→ 选 App IDs → 填写描述和 Bundle ID（建议反向域名格式）→ 勾选所需 Capabilities。
  4. 创建描述文件：Profiles → 「+」→ 选 App Store → 关联刚才的 App ID 和 Distribution Certificate → 下载 .mobileprovision，双击安装到 Xcode。
  5. 在 Xcode 中打开项目，Signing & Releases 配置中勾选「Automatically manage signing」并选择你的 Team，或手动选择对应 Profile。
💡 提示如果用 Xcode 的「Automatically manage signing」，Xcode 会自动帮你创建并管理证书和 Profile，对个人开发者最省心。组织账号建议手动管理以便团队协作。
来源：Apple Developer Account Help

3. 在 App Store Connect 创建 App 记录

证书配好后，去 App Store Connect 创建一条 App 记录。必须先有 App 记录才能上传构建版本。

3创建步骤

  1. 登录 App Store Connect（最初只有 Account Holder 可登录）。
  2. 接受协议：若要上架付费 App 或内购，先在「Business」中签署 Paid Applications Agreement，并填写银行和税务信息。免费 App 可跳过。
  3. 进入「我的 App」→ 点击「+」→「新建 App」。
  4. 填写：
    • 平台：iOS（游戏一般选 iOS）
    • 名称：App Store 显示名（30 字符以内，需未被占用）
    • 主要语言：简体中文等
    • Bundle ID：下拉选择第 2 步注册的 App ID
    • SKU：内部唯一标识（如 yourgame2026）
    • 用户访问权限：完全访问（默认）
  5. 点击「创建」，App 记录生成。
💡 协议先行付费协议和银行/税务信息如果没填，即使提审通过也无法上架收费项。免费游戏也建议尽早完成，避免日后接入内购时被卡。
来源：App Store Connect workflow

4. 填充应用资料

App 管理页面左侧有多个信息板块，提审前必须填齐。

① App 信息（App Information）

  • 名称 / 副标题：名称 30 字符，副标题 30 字符。
  • 主要类别：游戏选「Games」并选子类别。
  • 年龄分级：填写内容描述问卷，系统自动生成分级。iOS 26 新分级
  • Bundle ID / SKU / 主要语言：创建时已填，此处可微调。

② 版本信息

  • 截图与预览：必须提供至少 6.7 寸（iPhone 15 Pro Max）截图；可选 6.5"、iPad 等。App 预览视频可选（最多 3 个，15-30 秒）。
  • 描述：完整描述 4000 字符内；促销文本 170 字符（可随时改，无需重新提审）。
  • 关键词：100 字符以内，逗号分隔。
  • 支持 URL：必须提供有效的客服支持页面网址。
  • 审核信息：如有登录、付费墙，须提供演示账号给审核员。

③ 定价与销售范围

  • 设置价格层级（免费 / 付费层级）或订阅价格。选择上架的国家/地区（App Store 覆盖 175 个店面）。

④ App 隐私详情 必填

必须申报 App 的数据收集实践，包括第三方 SDK 的数据收集行为。不填写无法提审。

⚠️ 隐私申报是重灾区很多开发者因第三方 SDK（广告、分析）的数据收集行为未如实申报而被拒。接入任何 SDK 前先查清它收集什么数据，据实填写。
来源：Submitting apps — Apple Developer · App Privacy Details

5. 上传构建版本（Build）

⚠️ SDK 版本要求（2026 年现行）自 2026 年 4 月 28 日起，上传到 App Store Connect 的 iOS/iPadOS 应用须用 iOS 26 SDK 或更高版本构建（即需 Xcode 26）。

5上传方式

方式 A — Xcode（最常用）：

  1. Xcode 中选 Generic iOS Device 作为目标。
  2. 菜单 Product → Archive（归档）。
  3. 归档完成后弹出 Organizer，点「Distribute App」→「App Store Connect」→「Upload」。

方式 B — Transporter（macOS App）：导出 IPA 后用 Transporter 拖拽上传。

方式 C — Transporter 命令行 + JWT：用于自动化流水线。

上传成功后，构建版本需等待 Apple 处理（通常 15-30 分钟），处理完成后才能选用该版本提审。

来源：Upload builds — App Store Connect Help

6. TestFlight Beta 测试

类型内部测试外部测试
人数上限最多 100 人（App Store Connect 用户）最多 10000 人（邮件/公开链接邀请）
是否需审核否，立即可用首次需通过 Beta App Review
测试者要求须是 App Store Connect 团队成员任何人，需安装 TestFlight App

6操作步骤

  1. App 管理页 →「TestFlight」标签 → 选择已上传并通过处理的构建版本。
  2. 填写 Beta 测试信息：测试说明、反馈邮箱、隐私政策等。
  3. 内部测试：添加 App Store Connect 用户即可。
  4. 外部测试：创建测试组 → 添加测试者邮箱或开启公开链接 → 首次需等 Beta 审核。
来源：TestFlight — Apple Developer

7. 提交审核（Submit for Review）

7提审步骤

  1. App 管理页 → 选择对应版本 → 确认所有信息板块无红色缺失项。
  2. 在「构建版本」处点击选择已上传的 Build。
  3. 填写「导出合规信息」（加密声明）——大多数游戏选「不含加密」或符合豁免。
  4. 选择发布方式，点击右上角「添加以提审」→「提交审核」。

8. 发布上线

发布方式说明
手动发布审核通过后手动点击发布才上线。
分阶段发布 推荐7 天内按 1%→2%→5%→10%→20%→50%→100% 逐步推送，可随时暂停。

9. 发布后管理

  • 数据分析：App Analytics 查看展示量、下载量、留存、崩溃率。
  • 评价回复：可在 App Store Connect 回复用户评论。
  • 版本更新：创建新版本 → 上传新 Build → 重新提审。
  • 营收：Payments and Financial Reports 查看打款（月结）。
  • 推广：App 内活动（In-App Events）、自定义产品页、Featuring Nominations。

1注册步骤

  1. 用 Google 账号登录 play.google.com/console/signup。
  2. 选择账号类型：个人或组织。
  3. 填写开发者名称、联系邮箱、网站（可选）。

来源：Google Play Console 注册

2. 创建应用

2创建步骤

  1. 登录 Play Console → 右上角「Create app」。
  2. 填写：默认语言 + 应用名称、App 还是 Game（游戏选 Game）、免费还是付费、联系邮箱。
  3. 勾选同意「开发者计划政策」「美国出口法律」和 Play 应用签名服务条款。
  4. 点击「Create app」。
💡 包名永久唯一应用的 Package Name（如 com.yourgame.app）一旦创建即永久绑定，不可删除或复用。
来源：Create and set up your app — Play Console Help

3. 设置商店详情（Store Listing）

商品详情

  • 应用名称：最多 30 字符
  • 简短说明：最多 80 字符
  • 完整说明：最多 4000 字符

预览素材

  • 应用图标：512×512 px，32-bit PNG
  • 置顶大图：1024×500 px
  • 手机截图：最少 2 张，最多 8 张
  • 宣传视频：可添加 YouTube 链接

分类与标签

  • 游戏选 Games 并选子类别，最多 5 个标签。

多语言

  • 可添加多种语言的商店详情翻译。
来源：Store listing — Play Console Help

4. 应用内容与政策合规（App Content）必填

声明项说明
隐私政策 必填提供一个在线可访问的隐私政策 URL。
广告声明 App 是否含广告，含广告会显示「Contains ads」标签。
应用访问权限若有登录墙或受限区域，须提供演示账号和访问说明。
目标受众与内容声明目标年龄组。包含儿童的应用须遵守 Families 政策。
内容分级 必填填写分级问卷，获取 IARC 分级。未分级应用可能被下架。
数据安全 必填声明 App 收集/共享哪些数据、是否加密、用户能否删除数据等。
数据删除 必填提供用户账号删除机制，并声明相关链接/说明。
⚠️ 合规是 Google 下架重灾区数据安全表单和隐私政策务必如实填写，与实际 SDK 行为一致。虚假声明会导致应用暂停甚至封号。
来源：Prepare your app for review

5. 应用签名（Play App Signing）

Google Play 使用 Android App Bundle (.aab) 格式上传（2021 年 8 月起强制）。Google 通过 Play 应用签名机制托管你的应用签名密钥。

上传密钥
你用来给 AAB 签名的密钥，Google 用它验证你的身份。
应用签名密钥
Google 用来为分发的 APK 签名的密钥。

创建应用时接受 Play 应用签名服务条款即默认启用。

来源：Android App Bundles

6. 测试轨道（Testing Tracks）

轨道受众说明
内部测试最多 100 名受信任测试者最快，上传后几秒可用。
封闭测试 必经由你控制的测试者群组个人新号申请生产权限前的必经环节。
开放测试任何人可加入测试版在 Google Play 公开可见。需有生产权限后可用。
生产所有用户正式发布。

6创建测试版本

  1. 「Test and release」→ 选择轨道 →「Create release」。
  2. 填写版本名称，上传已签名的 AAB。
  3. 填写版本说明，添加测试者（邮箱列表或 Google 群组）。
  4. 审核发布。
来源：Set up tests — Play Console Help

7. 个人账号测试门槛（重要！）

⚠️ 2023.11 新规2023 年 11 月 13 日之后创建的个人开发者账号，发布首个 App 前必须满足封闭测试要求。
  • 至少 12 名测试者持续加入你的封闭测试。
  • 测试者须连续保持加入状态至少 14 天。
  • 满足后，在 Dashboard 点击「Apply for production」申请生产权限。
  • Google 审核申请，通常 7 天内出结果。
💡 招募建议从亲友、同事、同学、目标用户社群招募。鼓励测试者多使用功能并反馈。保留反馈记录——申请生产权限时可能需要总结。
来源：App testing requirements for new personal developer accounts

8. 生产发布（Production Release）

8发布步骤

  1. 确认商店详情、应用内容全部填写完成。
  2. 「Test and release」→「Production」→「Create release」。
  3. 填写版本名称，上传已签名的 AAB。
  4. 选择发布方式：标准发布或托管发布。
  5. 点击「Review release」→「Start rollout to Production」。
发布方式说明
标准发布提交审核通过后自动上线。
托管发布 推荐审核通过后不自动上线，待你手动批准发布，可精确控制上线时机。

分阶段发布

生产版本发布时可设置分阶段比例（如 10%、50%、100%），逐步扩大用户覆盖。

来源：Standard publishing · Managed publishing

9. 发布后管理

  • 数据分析：Play Console 的 Statistics、Android Vitals 查看安装、卸载、崩溃、ANR。
  • 评价管理：回复用户评论，查看评分分布。
  • 版本更新：创建新 Production release，支持分阶段更新。
  • 商店详情实验：A/B 测试不同图标、截图、说明的转化率。
  • 营收：财务报告按月出具，打款至关联的商户账号。

4. 构建上传与审核

构建上传（SteamPipe）

SteamPipe 是 Steam 的内容分发系统，支持两种上传方式：SteamPipe GUI（图形界面）和 SteamCMD + 构建脚本（命令行，适合 CI/CD）。

4上传与审核步骤

  1. 准备构建：确保包含所有必要的二进制文件、资源和 Steamworks SDK 依赖。
  2. 运行 SteamPipe 上传构建到对应 depot。
  3. 在 Steamworks 后台将构建与商店页面关联。
  4. 配置 Steamworks 功能：成就、云存档、排行榜等。
  5. 提交构建审核。Valve 会运行游戏、检查商店页面、确认配置正确且无危害行为。

审核阶段

  • 商店页面审核：检查名称、描述、素材是否符合规范。
  • 构建审核：Valve 人员运行你的游戏 1-2 次，检查能否启动、是否与描述一致、有无恶意行为。
  • 审核时长：通常 1-5 个工作日。
⚠️Steam 的构建审核并非全面的质量测试——Valve 不会帮你测 bug、验证玩法平衡。审核的重点是「不坑用户」，不是「游戏好不好玩」。
来源：Steamworks SDK Documentation

发布控制在你的手中——你决定何时按「发布」按钮。发布后可使用 Steamworks 更新构建、管理折扣、发起促销。

附录一：审核被拒常见原因

App Store 常见拒因

  • 2.1 App Completeness：应用崩溃、白屏、核心功能不可用。务必提供演示账号。
  • 2.3 Metadata：截图与实际不符、关键词堆砌、描述含其他平台名。
  • Guideline 4.3 Spam：与已有应用过度相似、低价值内容。
  • 5.1.1 Data Collection：隐私营养标签未如实申报第三方 SDK 数据收集。
  • 3.1.1/3.1.2 In-App Purchase：虚拟物品必须用 Apple 内购，不可跳转外部支付。

Google Play 常见拒因

  • 政策违规：数据安全表单与实际不符、权限滥用、误导性内容。
  • 目标 API 级别：未达到当前要求的 targetSdkVersion。
  • 隐私政策缺失或无效：URL 无法访问、内容不覆盖实际行为。
  • 内容分级未完成：未填问卷导致 Unrated。
  • 登录信息缺失：有登录墙但未提供审核员账号。

Steam 常见拒因

  • 商店页面问题：描述含误导信息、Capsule 图片不合规、素材缺失。
  • 构建问题：游戏无法正常启动、崩溃、与描述严重不符。
  • 内容红线：含仇恨言论、未授权版权内容、成人内容未标注。

Epic 常见拒因

  • 成就不对等：Steam 版有成就但 Epic 版缺失。
  • 跨平台联机缺失：在线多人游戏不支持跨 PC 商店联机。
  • IARC 分级缺失：韩国需额外 GRAC。

主机（PS/Xbox）常见拒因

  • 系统稳定性：崩溃、死机、过热——主机认证绝对零容忍。
  • TRC/XR 不合规：存档 API 使用不当、手柄重连异常、UI 适配问题。
  • Trophy/成就缺失或不完整。
  • 网络功能异常：离线时未正确处理、登录状态切换崩溃。
